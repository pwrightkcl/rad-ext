# Handoff Note - 2026-03-09

## Summary
- Investigated metadata processing failures in `index_dicom/index_dicom.py` on local `.dcm` files.
- User observed: from 1000 test files, 241 are MR, and 209 MR cases failed with:
  - `the truth value of an array is ambiguous`
- Root cause identified as null checks evaluating array-like values in boolean context.

## Code Changes Made
Updated file:
- `index_dicom/index_dicom.py`

Changes:
- Replaced unsafe null checks:
  - `value == "" or value is None or pd.isna(value)`
  - `value[0] == "" or value[0] is None or pd.isna(value[0])`
  - `if pd.isna(v):`
- Added and used scalar-safe helper:
  - `DicomIndexer._is_nullish(v: Any) -> bool`
- `_is_nullish` behavior:
  - Returns `True` for `None` and empty strings.
  - Calls `pd.isna(v)` inside `try/except`.
  - Converts result to bool only if scalar; catches `ValueError`/`TypeError` from array-like outputs and returns `False`.

## Why This Fix
- Some local MR DICOM metadata values are array-like but not `MultiValue`.
- `pd.isna(array_like)` returns array-like booleans; using that in `if` raises ambiguous truth-value error.
- PACS query path may not exhibit this because C-FIND responses are usually more limited/normalized than full local file metadata.
